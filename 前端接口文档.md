# 安全生产隐患 AI 自动识别项目前端接口文档

## 1. 文档说明

### 1.1 编写依据

- 页面原型：`页面原型/login.html`、`index.html`、`recognition.html`、`review.html`、`collection.html`、`dataset.html`、`dictionary.html`、`stats.html`

### 1.2 适用范围

- Web 管理端
- 前端技术栈：`Vue 3` + `Vite` + `Pinia`

## 2. 通用规范

### 2.1 基础信息

- 基础路径：`/api`
- 数据格式：`application/json`
- 文件上传：`multipart/form-data`
- 字符编码：`UTF-8`
- 时间格式：`yyyy-MM-dd HH:mm:ss`
- 日期格式：`yyyy-MM-dd`

### 2.2 统一响应结构

- `code = 1` 表示请求成功
- `code = 0` 或其他数字表示请求失败
- 请求成功时后端返回 `msg = null`
- 请求失败时后端返回 `msg`，前端展示后端返回的 `msg`

```json
{
  "code": 1,
  "msg": null,
  "data": {}
}
```

### 2.3 分页响应结构

```json
{
  "code": 1,
  "msg": null,
  "data": {
    "total": 0,
    "rows": []
  }
}
```

### 2.4 分页参数规范

- 分页参数统一使用 `page` 和 `pageSize`
- `page` 表示页码，默认值为 `1`
- `pageSize` 表示每页展示记录数，默认值为 `10`

### 2.5 通用 HTTP 失败响应示例

#### 401 未认证

```json
{
  "code": 401,
  "msg": "未登录或登录已过期",
  "data": null
}
```

#### 403 无权限

```json
{
  "code": 403,
  "msg": "无权访问该资源",
  "data": null
}
```

#### 404 资源不存在

```json
{
  "code": 404,
  "msg": "请求资源不存在",
  "data": null
}
```

#### 500 服务器异常

```json
{
  "code": 500,
  "msg": "服务器内部异常",
  "data": null
}
```

### 2.6 认证规范

- 认证框架：`Spring Security + JWT`
- 令牌机制：`accessToken + refreshToken`
- 请求头：`Authorization: Bearer {accessToken}`
- 登录成功后前端依次调用：`/auth/login`、`/auth/current-user`、`/auth/menus`
- 当前用户权限集合统一存入 `Pinia`
- 左侧菜单展示和页面访问统一根据 `menu:*` 权限码控制

### 2.7 菜单权限码

| 页面 | 权限码 |
| --- | --- |
| 首页仪表盘 | `menu:dashboard` |
| 图片识别 | `menu:recognition` |
| 人工复核 | `menu:review` |
| 数据采集 | `menu:collection` |
| 数据集管理 | `menu:dataset` |
| 隐患字典 | `menu:dictionary` |
| 统计分析 | `menu:stats` |

## 11. 前端工程规范

### 11.1 请求封装规范

- 请求库统一使用 `axios`
- 请求实例统一在 `src/utils/request.ts` 中创建
- `accessToken` 统一在请求拦截器中注入
- `401` 统一在响应拦截器中处理刷新令牌逻辑

### 11.2 Pinia 模块规范

| Store | 职责 |
| --- | --- |
| `useAuthStore` | 登录、退出、刷新令牌、当前用户信息 |
| `usePermissionStore` | 权限集合、菜单列表、路由过滤 |
| `useAppStore` | 布局状态、侧边栏状态、全局界面状态 |

### 11.3 路由权限规范

- 每个页面路由 `meta` 中声明 `menuPermission`
- 全局前置守卫先校验登录态，再校验 `menuPermission`
- 无页面权限时跳转 `403` 页面或首页
